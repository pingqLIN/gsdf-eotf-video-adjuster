# LumaLift 擴充介面設計參考

本檔是 Chrome 擴充控制面的專案設計來源，定義介面視覺語彙，以及外部參考與專案需求之間的邊界。

## 設計角色

LumaLift 是精簡的專業影片控制面板，不是 landing page。介面應該像可反覆使用的精密檢查儀表：資訊密度高、安靜、可讀，而且可信任。

首屏優先順序是操作：

- 啟用或旁路 EOTF/GSDF 效果
- 調整目標亮度、Gamma 補償與 filter amount
- 在基本、進階、診斷暫位頁籤之間切換且不中斷脈絡
- 僅透過右上角側邊欄控制開啟本專案的連續頻率解析度參考圖與即時曲線

## 參考邊界

Stitch 匯出內容只能作為視覺語彙參考，不是產品規格。

可以參考 Stitch 的部分：

- 字級密度與數值讀out層級
- 內凹 wells、細線邊框與上緣高光
- 緊湊的圖示按鈕與分段控制
- 深色儀表面板的對比節奏
- 標籤、滑桿、狀態列的間距節奏

不可採用 Stitch 的部分：

- 功能名稱或工作流程需求
- 圖表資料、公式行為或 GSDF 模型解讀
- 診斷圖樣內容
- 截圖影像作為正確性參考
- Stitch 裡的參考/測試圖；它不是本專案的連續頻率解析度圖

目前專案實作才是 GSDF-QC 圖樣的權威來源。連續頻率解析度、調變、線對、灰階與漸層檢查都由本機 canvas 程式產生，不得以 Stitch 錯誤參考圖取代。

## 視覺語彙

面板採用儀器控制面風格：

- 深色 obsidian 機身，而不是裝飾性玻璃效果
- 以 4px 為基礎的緊湊間距，同時保持可讀性
- 控制項 6px 圓角、外殼 8px 圓角；除了狀態指示外避免膠囊形控制
- 滑桿、預覽與群組控制使用內凹 control wells
- 用細微上緣高光與低對比分隔線建立層次，不依賴重陰影
- active state 使用克制的銀色；綠色只表示啟用/系統確認，青色只用於分析資料

字體分工：

- Bahnschrift 用於標籤、標題與緊湊 UI 控制
- Cascadia Mono 用於數值、單位、圖例與診斷標籤

互動控制的字距應保持克制。寬字距只保留給小型儀器標籤，不用於內文。

## 互動規則

純圖示按鈕必須保持方形、尺寸穩定，並透過 `title` 或 `aria-label` 說明用途。

分段控制必須讀起來像單一實體控制，不是分散的按鈕。啟用段落使用高對比金屬填色；未啟用段落保持內凹。

滑桿應像實體 fader：

- 6px 內凹軌道
- 矩形 thumb
- 需要時提供清楚 min/max 標籤
- 固定數值欄寬，避免數值更新造成版面跳動

完整 GSDF-QC 視圖必須保持內容正確。視覺打磨可以調整框架、標籤與控制外觀，但圖樣結構仍以本專案為準。

## 品質檢查

UI 變更完成前：

- 執行 extension 的 TypeScript/build 檢查
- 用實際瀏覽器確認渲染結果，不只看程式碼
- 若變更影響面板結構，檢查基本、進階、診斷，以及側邊欄開啟/關閉狀態
- 確認精簡寬度下文字不會與控制項碰撞
- 確認深色與淺色主題的按鈕邊界都清楚可見
